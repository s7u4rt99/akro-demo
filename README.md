# Akro Agent

Full-stack deep research app: **Next.js frontend** talks to a **FastAPI backend** that runs a multi-agent research pipeline (Planner → Researcher → Synthesizer → Critic). The backend always generates PDF and PPTX reports, writes them to `server/out`, and returns them in the API response so the frontend can preview and download.

## Architecture

- **Backend (Python)** — `server/` — FastAPI on **port 8000**
  - `POST /research` — run research; returns JSON (report + base64 PDF/PPTX) and writes PDF/PPTX to `server/out`
  - `POST /chat` — streamed chat for research Q&A (uses same OpenAI key)
  - `GET /health` — health check

- **Frontend (Next.js)** — `frontend/` — dev server on **port 3000**
  - Calls the backend at `NEXT_PUBLIC_BACKEND_URL` (default `http://localhost:8000`)
  - Research page: runs research via backend, shows report and PDF/PPTX preview/download
  - Chat panel: streams replies from backend `/chat`

## Prerequisites

- **Python 3.10+** (for backend)
- **Node.js 18+** (for frontend)
- **OpenAI API key** (backend + chat)
- **Tavily API key** (backend search)

## 1. Backend setup

All backend commands are run from the `server/` directory.

```bash
cd server

# Virtual env
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Install
pip install -e .

# Env (required)
cp .env.example .env
# Edit .env: set OPENAI_API_KEY and TAVILY_API_KEY (see server/.env.example)
```

Optional in `server/.env`: `OPENAI_MODEL` (default `gpt-4o-mini`).

## 2. Frontend setup

From the repo root:

```bash
cd frontend

# Install
npm install

# Env (optional — only if backend is not at localhost:8000)
cp .env.example .env
# Edit .env: set NEXT_PUBLIC_BACKEND_URL if needed (e.g. http://localhost:8000)
```

## 3. Run both (different ports)

Use two terminals.

**Terminal 1 — Backend (port 8000):**

```bash
cd server
source .venv/bin/activate
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

**Terminal 2 — Frontend (port 3000):**

```bash
cd frontend
npm run dev
```

- Frontend: [http://localhost:3000](http://localhost:3000)
- Backend API: [http://localhost:8000](http://localhost:8000)  
  - Docs: [http://localhost:8000/docs](http://localhost:8000/docs)

The frontend is configured to call the backend at `http://localhost:8000` by default (`NEXT_PUBLIC_BACKEND_URL`). If you run the backend on another host/port, set `NEXT_PUBLIC_BACKEND_URL` in `frontend/.env` (e.g. `http://127.0.0.1:8000` or `http://your-server:8000`).

## 4. How to use

1. Open [http://localhost:3000](http://localhost:3000).
2. Enter a research query and start the flow. The frontend calls `POST /research` on the backend.
3. When research completes:
   - The report is shown on the page.
   - PDF and PPTX are generated by the backend, written to `server/out/<slug>.pdf` and `server/out/<slug>.pptx`, and returned as base64. The frontend shows a PDF preview and Download PDF / Download PPTX.
4. Use **Chat** to ask follow-up questions; the frontend streams from `POST /chat` on the backend.

## Output files

The backend always writes:

- `server/out/<query_slug>.pdf`
- `server/out/<query_slug>.pptx`

The same files are also returned in the `/research` JSON response as `pdf_base64` and `pptx_base64` so the frontend can preview and download without extra requests.

More detail on the research pipeline and API: see **server/README.md**.
